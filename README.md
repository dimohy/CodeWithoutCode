# 바이브 코딩을 위한 컴퓨팅 사고 시리즈

AI와 함께하는 새로운 프로그래밍 패러다임, **바이브 코딩**을 배우는 15주차 강의 시리즈입니다.

---

## 시리즈 소개

이 시리즈는 코딩을 직접 작성하지 않고도 문제를 해결할 수 있는 "바이브 코딩"을 다룹니다. GitHub Copilot과 함께 사고하고 협업하는 방법을 배우며, 컴퓨팅 사고의 4대 원리(분해, 패턴 인식, 추상화, 알고리즘적 사고)를 체계적으로 학습합니다.

바이브 코딩은 단순히 AI 도구를 사용하는 것을 넘어, **어떻게 생각하고 표현할 것인가**에 집중하는 새로운 프로그래밍 패러다임입니다. 여러분은 이 책을 통해 복잡한 문제를 논리적으로 분해하고, 명확하게 표현하며, GitHub Copilot과 효과적으로 협업하는 방법을 배우게 됩니다.

---

## 시리즈 구성

### 1권: [생각을 코드로](생각을%20코드로/) - 누구나 할 수 있는 바이브 코딩 사고

**대상 독자**: 프로그래밍 경험이 없거나 적은 일반인

코딩 경험이 없어도 괜찮습니다. 이 책은 여러분이 이미 일상에서 사용하고 있는 문제 해결 능력을 컴퓨팅 사고로 발전시켜 드립니다. 전문 용어 대신 쉬운 일상 언어로 설명하며, GitHub Copilot이라는 친구와 함께 자연스럽게 문제를 해결하는 방법을 배웁니다.

**학습 목표**:
- 컴퓨팅 사고의 4대 원리 이해 및 적용
- GitHub Copilot과의 효과적인 대화 방법 습득
- 실생활 문제를 바이브 코딩으로 해결하는 능력 배양
- GitHub Copilot Agent 모드 중급 수준 활용

**주요 특징**:
- 코드를 읽지 않아도 되는 학습 방식
- 일상 속 사례로 쉽게 이해하는 개념 설명
- 따뜻하고 공감적인 문체
- 단계별 실습과 프롬프트 작성 연습

### 2권: 사고의 깊이 - 전문가를 위한 바이브 코딩과 컴퓨팅 사고

**대상 독자**: 프로그래밍 경험이 있는 개발자 및 IT 전문가

이미 코딩을 알고 계신가요? 이 책은 여러분의 개발 방식을 한 단계 끌어올립니다. 대규모 시스템 설계부터 레거시 코드 현대화까지, GitHub Copilot과 협업하여 더 빠르고 효율적으로 문제를 해결하는 방법을 배웁니다.

**학습 목표**:
- 고급 컴퓨팅 사고 및 시스템 설계 능력 향상
- 복잡한 프로젝트를 위한 고급 프롬프트 엔지니어링
- GitHub Copilot을 활용한 코드 생성, 리뷰, 리팩토링
- 대규모 시스템 아키텍처 설계 및 최적화
- GitHub Copilot Agent 모드 중급 수준 활용

**주요 특징**:
- 생성된 코드를 빠르게 검증하고 개선하는 기법
- 실무 중심의 아키텍처 설계 및 리팩토링 전략
- 품질 관리 및 윤리적 고려사항
- 산업별 적용 사례 분석

## 책의 구조

각 책은 15주차 대학교 강의 형식으로 구성되어 있습니다.

### 주차별 구성

각 챕터는 다음과 같은 구조로 이루어져 있습니다:

1. **개요**: 챕터의 학습 목표와 전체 흐름 소개
2. **개념 설명**: 핵심 개념과 이론적 배경 설명
3. **구체적인 설명**: 실제 적용 방법과 세부 사항
4. **실습**: GitHub Copilot을 활용한 단계별 실습
5. **실습 결과 요약**: 학습 내용 정리 및 핵심 포인트 강조

### 학습 흐름

**일반인편 (생각을 코드로)**:
- 1-3주차: 바이브 코딩과 컴퓨팅 사고 소개
- 4-5주차: 컴퓨팅 사고 4대 원리 학습
- 6주차: GitHub Copilot과의 협업 방법
- 7-9주차: 실습 및 4대 원리 종합 연습
- 10-11주차: 실생활 문제 해결
- 12-13주차: 최종 프로젝트
- 14-15주차: 평가 및 미래 전망

**전문가편 (사고의 깊이)**:
- 1주차: 프로그래밍 패러다임의 전환
- 2주차: 컴퓨팅 사고 4대 원리 집중 학습
- 3-6주차: 고급 컴퓨팅 사고 및 시스템 설계
- 7주차: 중간평가
- 8-11주차: 실전 적용 및 윤리
- 12-15주차: 최종 프로젝트 및 평가

## 이렇게 읽으세요

### 일반인편을 읽으시는 분들께

**코드를 보지 않아도 됩니다**. 이 책은 코드를 읽거나 이해하는 것이 아니라, **무엇을 요청할지** 생각하고 **어떤 결과를 얻었는지** 확인하는 데 집중합니다.

1. **순서대로 읽으세요**: 각 챕터는 이전 내용을 기반으로 구성되어 있습니다.
2. **실습을 반드시 따라하세요**: 읽기만 해서는 부족합니다. GitHub Copilot과 직접 대화하며 경험을 쌓으세요.
3. **완벽을 추구하지 마세요**: 처음부터 완벽한 프롬프트를 작성할 필요는 없습니다. 시행착오도 학습 과정의 일부입니다.
4. **여러분의 언어로 표현하세요**: 프로그래밍 용어를 몰라도 괜찮습니다. 일상 언어로 문제를 설명하는 연습을 하세요.
5. **천천히, 하지만 꾸준히**: 한 주에 한 챕터씩, 15주 동안 꾸준히 학습하는 것을 권장합니다.

### 전문가편을 읽으시는 분들께

**생성된 코드를 빠르게 검증하세요**. 이 책은 GitHub Copilot이 생성한 코드를 효과적으로 검토하고 개선하는 방법을 가르칩니다.

1. **선택적으로 읽어도 됩니다**: 이미 알고 있는 개념은 건너뛰고 필요한 부분만 읽어도 괜찮습니다.
2. **실습에 집중하세요**: 이론보다는 실제 적용 사례와 프롬프트 패턴을 익히는 데 집중하세요.
3. **자신의 프로젝트에 적용하세요**: 학습한 내용을 바로 실무에 적용해 보세요.
4. **코드 품질을 놓치지 마세요**: GitHub Copilot이 생성한 코드를 맹신하지 말고 항상 검증하세요.
5. **자신만의 워크플로우를 만드세요**: 책의 내용을 기반으로 여러분에게 맞는 협업 패턴을 개발하세요.

### 두 책을 모두 읽으시는 분들께

일반인편부터 시작하여 기초를 탄탄히 다진 후, 전문가편으로 넘어가는 것을 권장합니다. 일반인편에서 배운 컴퓨팅 사고의 기초가 전문가편의 고급 내용을 이해하는 데 큰 도움이 됩니다.

## GitHub Copilot 준비하기

이 책의 모든 실습은 GitHub Copilot을 사용합니다. 학습을 시작하기 전에 다음을 준비하세요:

1. **GitHub 계정 생성**: [github.com](https://github.com)에서 무료 계정을 만드세요.
2. **GitHub Copilot 구독**: 개인 사용자는 월 $10, 학생은 무료로 이용할 수 있습니다.
3. **Visual Studio Code 설치**: GitHub Copilot을 사용할 수 있는 무료 코드 에디터입니다.
4. **GitHub Copilot 확장 설치**: VS Code에서 GitHub Copilot 확장을 설치하세요.

자세한 설정 방법은 각 책의 3주차에서 단계별로 안내합니다.
