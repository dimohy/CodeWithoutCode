# 7주차: 실습 - 분해와 패턴 인식 연습

## 개요

### 지난 주 복습: Agent와 대화하는 법 배움

지난 6주차에서 우리는 GitHub Copilot Agent와 효과적으로 대화하는 방법을 배웠습니다. 명확하고 구체적으로 요청하기, 목적을 먼저 말하기, 맥락 제공하기, 한 번에 하나씩 요청하기 같은 핵심 원칙들을 익혔습니다.

또한 프롬프트 엔지니어링의 기초도 학습했습니다. 좋은 프롬프트는 문제 정의, 예상 결과 명시, 제약 조건이라는 3요소를 포함한다는 것을 배웠죠. 그리고 Agent의 응답이 만족스럽지 않을 때 어떻게 개선을 요청할지, 반복적 대화를 통해 점차 원하는 결과에 가까워지는 방법도 익혔습니다.

이제 우리는 **사고하는 법**(4대 원리)과 **전달하는 법**(Agent와의 대화)을 모두 배웠습니다. 하지만 아직 책으로 만 알고 있을 뿐이죠.

### 이번 주: 이론에서 실천으로

지금까지 배운 컴퓨팅 사고의 원리들을 이제 본격적으로 실습할 시간입니다. 이번 주는 4대 원리 중 처음 두 가지인 **분해**와 **패턴 인식**에 집중합니다. 이론으로 배운 것과 실제로 적용하는 것은 큰 차이가 있습니다. 직접 해보면서 체득하는 시간이 될 것입니다.

분해는 복잡한 문제를 작고 다루기 쉬운 조각으로 나누는 기술입니다. 패턴 인식은 그 조각들 속에서 반복되는 규칙이나 유사성을 찾아내는 능력입니다. 이 두 가지를 잘 활용하면 어떤 복잡한 문제도 해결의 실마리를 찾을 수 있습니다.

오늘 실습에서는 실생활의 구체적인 예제들을 다룹니다. 아침 준비 루틴처럼 매일 하는 일부터 가계부 데이터 정리까지, 여러분이 실제로 마주하는 문제들입니다. 이런 친숙한 예제로 시작하면 개념이 훨씬 쉽게 와닿을 것입니다.

이번 주차가 끝나면 여러분은:
- 복잡한 문제를 효과적으로 분해할 수 있습니다
- 데이터나 과정에서 패턴을 발견할 수 있습니다
- 분해와 패턴 인식을 GitHub Copilot에게 전달할 수 있습니다
- 다양한 상황에서 이 원리들을 응용할 수 있습니다

실습 중심이므로 많이 시도하고 경험하는 것이 중요합니다. 틀려도 괜찮습니다. 시행착오 자체가 배움입니다.

### 이번 주 실습의 구조

이번 실습은 단계적으로 진행됩니다:

**1부: 분해 연습하기**
- 일상의 익숙한 활동들을 단계별로 나누어봅니다
- 좋은 분해와 나쁜 분해를 비교하며 기준을 배웁니다
- 실제로 여러분의 루틴을 분해해보는 과제를 합니다

**2부: 패턴 찾기**
- 가계부, 할 일 목록 같은 실제 데이터에서 규칙을 발견합니다
- 시간, 공간, 구조, 행동 패턴을 구분하는 법을 배웁니다
- 발견한 패턴을 어떻게 활용할지 생각해봅니다

**3부: GitHub Copilot에게 전달하기**
- 분해한 내용을 프롬프트로 작성하는 법을 배웁니다
- 패턴을 기능 요청으로 바꾸는 연습을 합니다
- 효과적인 프롬프트 템플릿을 익힙니다

### 실습하는 방법

각 예제를 볼 때는 이렇게 해보세요:

1. **먼저 스스로 생각하기** - 예시를 보기 전에 어떻게 할지 종이에 적어보세요
2. **예시와 비교하기** - 여러분의 방법과 예시가 어떻게 다른지 살펴보세요
3. **왜 그렇게 했는지 이해하기** - 예시의 선택에는 이유가 있습니다
4. **직접 적용하기** - 비슷한 다른 주제로 연습해보세요

틀려도 괜찮습니다. 중요한 것은 "왜 이렇게 나눴지?", "이 패턴이 의미 있는 걸까?"라고 계속 질문하는 것입니다.

준비되셨나요? 그럼 첫 번째 실습부터 시작해봅시다!

## 일상 문제를 작은 단계로 나누기

### 분해의 실전 적용

분해는 "큰 문제를 작은 문제로"라는 간단한 원리지만, 실제로 적용하려면 연습이 필요합니다. 처음에는 어디서 나눌지, 얼마나 작게 나눌지 막막하게 느껴질 수 있습니다. 하지만 걱정하지 마세요. 여러분은 이미 일상에서 이런 방식으로 생각하고 있습니다. 다만 의식하지 못했을 뿐이죠.

예를 들어, "저녁 준비"라는 큰 일을 떠올려보세요. 여러분은 자연스럽게 "무엇을 먹을지 정하기", "재료 준비하기", "요리하기", "설거지하기"로 나눠서 생각합니다. 이것이 바로 분해입니다. 이제 이 자연스러운 능력을 좀 더 체계적으로 사용하는 법을 배울 것입니다.

어떤 분해가 좋은 분해일까요? 네 가지 기준을 기억하면 됩니다. 각 부분이 서로 겹치지 않고 독립적이어야 하고(독립성), 모든 부분을 합치면 전체가 되어야 하며(완결성), 각 부분이 너무 크지도 작지도 않아야 하고(적절한 크기), 이름만 봐도 무엇을 하는지 알 수 있어야 합니다(명확성). 복잡하게 들리나요? 예제를 보면 금방 이해가 될 것입니다.

**실습 1: 아침 준비 루틴 분해하기**

아침에 일어나서 출근하기까지의 과정을 분해해봅시다. 단순해 보이지만, 막상 단계로 나누려면 "어디까지 나눠야 하지?"라는 고민이 생깁니다. 이것이 자연스러운 반응입니다. 완벽한 분해는 없으니까요. 상황과 목적에 따라 다르게 나눌 수 있습니다.

**먼저 여러분이 해보세요**

예시를 보기 전에, 잠깐 시간을 내어 여러분의 아침 루틴을 종이나 메모장에 적어보세요. 알람이 울린 후 무엇을 하시나요? 몇 단계로 나눌 수 있을까요? 어떤 순서로 하시나요? 5분 정도만 투자해보세요. 정답을 찾으려고 하지 마세요. 여러분만의 방식으로 나누면 됩니다. 나중에 예시와 비교하면서 "아, 이렇게도 나눌 수 있구나"라는 깨달음을 얻는 것이 중요합니다.

---

이제 함께 단계별로 분해해봅시다.

**1단계: 전체 과정 파악하기**

먼저 큰 그림을 봅니다. 무엇을 시작점으로 하고, 무엇을 끝점으로 할지 명확히 정하는 것입니다.

```
목표: 아침에 일어나서 제시간에 출근 준비 완료하기
시작: 알람 소리
끝: 현관문을 나섬
```

목표를 이렇게 명확히 하면 어디서 시작하고 끝나는지가 분명해집니다. "준비"라는 막연한 표현보다 "현관문을 나설 때까지"라고 하면 훨씬 구체적이죠.

**2단계: 주요 활동 나열하기**

이제 큰 덩어리로 나눕니다. 너무 세세하게 들어가지 않고, 주요 카테고리만 생각합니다.

1. 기상 및 몸 깨우기
2. 씻기
3. 옷 입기
4. 아침 식사
5. 준비물 챙기기
6. 외출 준비

보시다시피 6개의 큰 활동으로 나눴습니다. 사람마다 다를 수 있습니다. 어떤 분은 "운동하기"를 추가할 수도 있고, 어떤 분은 아침을 안 먹어서 4번을 생략할 수도 있습니다. 그게 자연스럽습니다. 여러분의 루틴에 맞게 나누면 됩니다.

**3단계: 각 활동을 세부 단계로 분해하기**

더 구체적으로 나눕니다.

**기상 및 몸 깨우기:**
- 알람 끄기
- 침대에서 일어나기
- 스트레칭하기
- 커튼 열어 햇빛 받기

**씻기:**
- 화장실 가기
- 세수하기
- 양치하기
- 샤워하기 (선택)

**옷 입기:**
- 오늘 입을 옷 고르기
- 속옷 입기
- 겉옷 입기
- 신발 신기 (외출 직전)

**아침 식사:**
- 메뉴 정하기
- 음식 준비하기 (또는 데우기)
- 먹기
- 설거지하기

**준비물 챙기기:**
- 가방 확인
- 지갑, 휴대폰, 열쇠 챙기기
- 필요한 서류나 물건 챙기기

**외출 준비:**
- 가스, 전등 확인
- 창문 잠금 확인
- 현관문 잠그기

**4단계: GitHub Copilot에게 전달하기**

이제 이 분해를 활용하여 아침 루틴 체크리스트 앱을 만들어달라고 요청할 수 있습니다.

```
GitHub Copilot에게:

"아침 준비 체크리스트 앱을 만들고 싶습니다.

6개의 카테고리가 있습니다:
1. 기상 (알람 끄기, 일어나기, 스트레칭, 커튼 열기)
2. 씻기 (화장실, 세수, 양치, 샤워)
3. 옷 입기 (옷 고르기, 입기)
4. 아침 식사 (준비하기, 먹기, 설거지)
5. 준비물 (가방, 지갑, 휴대폰, 열쇠)
6. 외출 준비 (가스/전등 확인, 창문, 현관문)

각 항목을 체크할 수 있고, 카테고리별로 진행 상황을 
볼 수 있으면 좋겠습니다."
```

**왜 이렇게 분해했을까요?**

위의 분해를 보면서 여러분은 어떤 생각이 드시나요? "나는 이렇게 안 나눴는데..."라고 생각하셨을 수도 있습니다. 그래도 괜찮습니다. 중요한 것은 여러분의 방식과 이 예시를 비교하면서 "왜 이렇게 나눴을까?"를 생각해보는 것입니다.

이 분해에는 몇 가지 원칙이 녹아 있습니다. 각 카테고리는 서로 겹치지 않습니다. "씻기"와 "옷 입기"는 명확히 다른 활동이죠. 물론 순서는 있습니다(옷 입기 전에 씻기). 하지만 각각이 독립적인 활동이라는 것이 중요합니다. 이것이 **독립성**입니다.

그리고 모든 카테고리를 완료하면 출근 준비가 끝납니다. 빠진 부분이 없습니다. 이것이 **완결성**입니다. 만약 "가방 챙기기"를 빠뜨렸다면? 현관문을 나서서 "어, 가방!"하고 돌아와야겠죠.

"양치하기"는 한 단계로 충분합니다. 이것을 "칫솔에 치약 짜기", "치아 앞면 닦기", "치아 뒷면 닦기"로 나누면 어떨까요? 너무 세세합니다. 우리의 목적은 아침 루틴 체크리스트를 만드는 것이지, 양치 교육 자료를 만드는 것이 아니니까요. 이것이 **적절한 크기**입니다.

마지막으로, "씻기"라는 단어만 봐도 무엇을 하는지 알 수 있습니다. "준비 1", "준비 2" 같은 이름보다 훨씬 명확하죠. 이것이 **명확성**입니다.

**다른 방법은 없을까요?**

당연히 있습니다! 분해에 정답은 없습니다. 여러분은 이렇게 나눴을 수도 있습니다:

```
방법 2: 시간 순서대로
- 6:00 - 기상 및 씻기
- 6:30 - 옷 입고 아침 먹기
- 7:00 - 준비물 챙기고 나가기
```

이것도 좋은 분해입니다! 시간을 기준으로 나눈 것이죠. 어떤 방법을 선택할지는 **목적**에 달려 있습니다. 체크리스트 앱을 만든다면 활동 기준 분해가 좋습니다. 시간 관리 앱을 만든다면 시간 기준 분해가 낫겠죠. 준비물 확인 앱이라면 장소 기준으로 나눌 수도 있습니다(침실에서 할 일, 화장실에서 할 일, 부엌에서 할 일, 현관에서 할 일).

분해는 도구입니다. 망치로 못도 박고 나사도 박을 수는 없듯이, 목적에 맞는 분해 방식을 선택해야 합니다. 하나의 정답을 찾으려고 하지 마세요. 여러분의 상황에 맞는 방식을 찾는 것이 중요합니다.

**실습 과제 1: 여러분의 루틴 분해하기**

이제 여러분 차례입니다. 다음 중 하나를 선택하여 분해해보세요:

**선택 1: 저녁 루틴**
퇴근하거나 하교한 후부터 잠들기까지의 과정을 분해해보세요. 평일 저녁은 어떤 활동들로 채워지나요? 2-3단계 계층으로 나누어 주요 카테고리와 세부 활동을 구분해보세요.

**선택 2: 주말 아침**
평일과는 다른 여러분만의 주말 아침 루틴을 생각해보세요. 어떤 점이 평일과 다른가요? 좀 더 여유롭게 하는 활동이 있나요? 그것들을 포함하여 분해해보세요.

**선택 3: 운동 루틴**
운동을 시작해서 마무리하기까지의 과정을 나눠보세요. 준비 운동, 본 운동, 정리 운동으로 크게 나눈 다음, 각 단계의 세부 활동을 작성해보세요.

**체크리스트**로 확인하세요:
- ✅ 2-3단계 계층으로 나눴나요?
- ✅ 각 단계가 서로 독립적인가요?
- ✅ 빠진 부분은 없나요?
- ✅ 너무 세세하거나 너무 크진 않나요?
- ✅ 단계 이름이 명확한가요?

5-10분 정도 시간을 들여 종이나 메모장에 작성해보세요. 완벽하지 않아도 됩니다. 나중에 수정할 수 있으니까요. 중요한 것은 직접 해보는 것입니다. 머릿속으로만 생각하는 것과 종이에 적어보는 것은 완전히 다른 경험입니다.

**실습 2: 온라인 쇼핑 과정 분해하기**

**과제 1: 운동 루틴 계획하기**

운동을 시작하기로 결심했다고 상상해보세요. 막연히 "운동해야지"라고 생각하는 것과, 구체적인 단계로 나눠서 생각하는 것은 완전히 다릅니다. 어떤 단계들이 필요할까요? 예시를 보기 전에 5분 정도 시간을 내어 종이에 적어보세요. 처음부터 완벽할 필요는 없습니다.

**예시 분해:**

```
목표: 규칙적인 운동 습관 만들기

1단계: 준비 단계
- 운동 목표 정하기 (체중 감량, 근력 강화, 건강 유지 등)
- 현재 체력 수준 파악하기
- 운동 장소 선택하기 (집, 헬스장, 야외)
- 필요한 장비 준비하기

2단계: 계획 수립
- 주당 운동 일수 정하기 (예: 주 3회)
- 운동 시간대 정하기 (아침/저녁)
- 운동 종류 선택하기
  - 유산소 (달리기, 자전거)
  - 근력 (웨이트, 맨몸 운동)
  - 유연성 (스트레칭, 요가)
- 주차별 목표 설정 (점진적 증가)

3단계: 실행
- 워밍업 (5-10분)
  - 가벼운 스트레칭
  - 몸풀기 운동
- 본 운동 (30-40분)
  - 계획한 운동 실행
  - 강도 조절
  - 휴식 시간 조절
- 쿨다운 (5-10분)
  - 스트레칭
  - 호흡 정리

4단계: 기록 및 평가
- 운동 내용 기록
  - 날짜, 시간
  - 운동 종류
  - 세트, 횟수, 무게
  - 소요 시간
- 체감 강도 평가 (1-10점)
- 다음 운동 시 조정 계획

5단계: 지속 및 개선
- 주간 진행 상황 점검
- 목표 대비 실행률 확인
- 어려운 점 파악 및 해결
- 운동 계획 수정 (필요시)
```

보시다시피 5단계로 나눴습니다. 준비, 계획, 실행, 평가, 개선이라는 흐름이 보이시나요? 이것은 비단 운동뿐 아니라 많은 활동에 적용되는 패턴입니다. 새로운 것을 시작할 때는 대부분 이런 흐름을 따릅니다.

특히 주목할 점은 "실행" 단계가 워밍업, 본 운동, 쿨다운으로 세분화되어 있다는 것입니다. 많은 초보자들이 워밍업과 쿨다운을 건너뛰는데, 이것들을 분해 과정에 명시적으로 포함시키면 잊지 않고 실행하게 됩니다. 이것이 분해의 힘입니다.
```

### 분해 실습에서 배운 점

세 가지 예제를 보면서 무엇을 발견했나요? 어쩌면 처음에 여러분이 생각했던 것과 많이 달라서 당황하셨을 수도 있습니다. 혹은 "아, 나도 비슷하게 생각했는데!"라며 뿌듯함을 느끼셨을 수도 있습니다. 어떤 반응이든 괜찮습니다. 중요한 것은 이 과정에서 배우는 것입니다.

**같은 원리, 다른 적용**

흥미로운 사실을 발견하셨나요? 운동, 여행, 글쓰기는 완전히 다른 활동입니다. 하나는 몸을 쓰는 것이고, 하나는 여가 활동이고, 하나는 창작 활동입니다. 하지만 분해 방식을 보면 비슷한 패턴이 보입니다. 준비 단계가 있고, 계획 단계가 있고, 실행 단계가 있고, 평가나 정리 단계가 있습니다.

이것은 우연이 아닙니다. 대부분의 목표 지향적 활동은 이런 흐름을 따릅니다. 일단 이 패턴을 알고 나면, 완전히 새로운 문제를 만나도 당황하지 않게 됩니다. "음, 준비는 뭐가 필요하지? 계획은 어떻게 세우지?"라고 자연스럽게 생각하게 됩니다.

**세부 수준은 목적에 따라 달라집니다**

운동 루틴은 5단계로 충분했는데, 여행 계획은 6단계, 글쓰기는 7단계로 나눴습니다. 왜 이렇게 차이가 날까요? 운동은 상대적으로 반복적이고 패턴화된 활동입니다. 한번 루틴을 정하면 비슷하게 반복됩니다. 하지만 여행은 준비할 것도 많고 예외 상황도 고려해야 합니다. 글쓰기는 창의적인 과정이 들어가서 단계가 더 세분화됩니다.

무엇이 더 나은 분해일까요? 정답은 없습니다. 복잡한 활동은 더 세세하게 나누고, 단순한 활동은 간단하게 나누면 됩니다. 목적에 맞게 조정하는 것이 핵심입니다.

**분해는 생각을 정리하는 마법입니다**

막연히 "여행 가야지"라고 생각할 때와, 6단계로 나눠서 "1단계: 기획부터 시작하자"라고 생각할 때를 비교해보세요. 어느 쪽이 더 실행하기 쉬울까요? 당연히 후자입니다. 분해는 막연함을 구체성으로 바꿉니다. "글을 써야 하는데 막막하다"가 "일단 주제부터 정하자"로 바뀌면, 첫발을 떼기가 훨씬 쉬워집니다.

### 패턴 인식의 실전 적용

이제 분해의 짝꿍인 패턴 인식으로 넘어가봅시다. 분해가 "나누는 것"이었다면, 패턴 인식은 "연결하는 것"입니다. 겉보기에 무관해 보이는 것들 사이에서 숨겨진 규칙을 찾아내는 능력이죠.

패턴은 항상 눈에 보이지 않습니다. 그래서 어렵게 느껴집니다. 데이터를 한두 번 보는 것만으로는 잘 보이지 않아요. 시간을 두고, 다양한 각도에서 바라봐야 합니다. 마치 숨은그림찾기처럼요. 한번 보이기 시작하면, 그 다음부터는 쉽게 보입니다.

패턴에는 여러 종류가 있습니다. 시간에 따라 반복되는 **시간적 패턴**, 위치나 배치에서 나타나는 **공간적 패턴**, 형태나 구조가 비슷한 **구조적 패턴**, 그리고 행동이나 과정에서 반복되는 **행동 패턴**이 있습니다. 각각의 특징을 알면 패턴을 더 쉽게 발견할 수 있습니다.

**실습 3: 가계부 데이터에서 패턴 찾기**

한 달간의 지출 데이터가 있다고 가정합시다.

**데이터 예시:**
```
11/1 - 점심 (식비) - 8,000원
11/1 - 버스 (교통비) - 1,250원
11/2 - 점심 (식비) - 7,500원
11/2 - 커피 (식비) - 4,500원
11/3 - 점심 (식비) - 9,000원
11/3 - 버스 (교통비) - 1,250원
11/4 - 영화 (여가) - 14,000원
11/4 - 저녁 (식비) - 25,000원
...
```

**패턴 찾기:**

**시간적 패턴:**
- 평일에는 점심 식비가 거의 매일 발생
- 버스비는 출근하는 날에만 발생
- 주말에는 여가비가 증가
- 금요일 저녁 지출이 평소보다 많음

**금액 패턴:**
- 점심 식비: 대부분 7,000~9,000원 사이
- 버스비: 항상 1,250원
- 커피: 주로 4,000~5,000원
- 주말 외식: 20,000원 이상

**카테고리 패턴:**
- 식비가 전체의 약 60%
- 교통비는 일정하게 유지
- 여가비는 주말에 집중

**행동 패턴:**
- 커피는 오후에 주로 구매
- 주중에는 집에서 식사, 주말에는 외식
- 월말에 지출이 줄어듦 (예산 소진)

**GitHub Copilot에게 패턴 활용하기:**

```
"가계부 앱을 만들고 싶습니다.

사용자의 지출 패턴을 분석하여:
1. 카테고리별 지출 비율을 보여주세요
2. 자주 발생하는 지출 항목을 표시해주세요
3. 평균 지출 금액을 계산해주세요
4. 예상과 다른 지출이 있으면 알려주세요

예를 들어:
- 평일 점심은 보통 8,000원인데 오늘은 15,000원이면 표시
- 매일 나가던 커피값이 오늘은 없으면 물어보기
- 이번 달 식비가 평소보다 많으면 경고"
```

**패턴 예시:**

**반복 패턴:**
- 매일: 운동, 독서, 일기 쓰기
- 매주 월요일: 회의 준비
- 매주 금요일: 주간 보고서
- 매월 말일: 정리 및 계획

**시간대 패턴:**
- 아침: 중요하고 집중 필요한 일
- 점심 후: 가벼운 업무나 미팅
- 저녁: 정리 작업

**우선순위 패턴:**
- 마감일이 가까운 것이 우선순위 높음

독서 기록 관리 앱을 만들고 싶습니다.

[분해]
독서 과정을 다음과 같이 나눴습니다:
1. 책 선정 (읽고 싶은 책 목록)
2. 독서 중 (현재 읽는 책, 진행률)
3. 완독 (다 읽은 책, 별점, 한줄평)
4. 통계 (월별 독서량, 장르별 선호도)

[패턴]
제 독서 패턴:
- 한 번에 2-3권 병행해서 읽음
- 주말에 주로 읽음
- 완독까지 평균 2주 소요
- 소설을 가장 많이 읽음 (60%)

[기능]
이 패턴을 활용하여:
1. 읽는 책이 3권 이상이면 '너무 많아요' 경고
2. 1주일 진행이 없으면 '책 읽으실 시간이세요?' 알림
3. 같은 장르가 계속되면 다른 장르 추천
4. 월 목표 권수 제안 (평균 기준)

웹 페이지로 만들어주세요.
```

**예시 2: 운동 루틴 추적기**

```
운동 루틴을 기록하고 패턴을 분석하는 앱이 필요합니다.

[분해]
운동 과정:
1. 준비 (스트레칭 5분)
2. 본 운동 (종목별로 세트 수행)
3. 마무리 (쿨다운 5분)
4. 기록 (운동 내용, 시간, 느낌)

각 종목:
- 이름 (예: 스쿼트, 플랭크)
- 세트 수
- 반복 횟수 (또는 시간)
- 휴식 시간

[패턴]
제 운동 패턴:
- 월/수/금: 상체 운동
- 화/목: 하체 운동
- 각 운동 30-40분 소요
- 점진적으로 무게 증가 (주당 +1kg)

[기능]
1. 요일별 운동 프로그램 자동 제안
2. 이전 기록 기반으로 무게/횟수 제안
   (예: '지난번 스쿼트 50kg 10회 성공. 
   오늘은 51kg 10회 시도해볼까요?')
3. 휴식이 너무 길면 알림
4. 주간 진행 상황 요약
5. 근력 향상 그래프

간단하고 사용하기 쉽게 만들어주세요.
```

**예시 3: 습관 형성 앱**

```
좋은 습관을 만들기 위한 추적 앱입니다.

[분해]
습관 형성 과정:
1. 습관 설정 (무엇을, 언제, 어떻게)
2. 실행 (매일 체크)
3. 분석 (연속 일수, 성공률)
4. 보상 (목표 달성 시)

[패턴]
습관 형성 패턴 (연구 기반):
- 초기 3일이 가장 어려움
- 21일 지속하면 습관화
- 주말에 실패 확률 높음
- 아침 습관이 성공률 높음

[기능]
1. 초기 3일 집중 격려
   (예: '가장 중요한 3일입니다. 화이팅!')
2. 연속 일수 표시 (동기 부여)
3. 위험 시기 미리 알림
   (예: '금요일 저녁, 평소 실패하기 쉬운 시간이에요')
4. 친구와 함께 도전 (선택사항)
5. 21일 달성 시 축하 메시지

동기부여가 되는 디자인으로 만들어주세요.
```

## 프롬프트 작성 패턴 연습

### 효과적인 프롬프트 템플릿

분해와 패턴 인식을 프롬프트에 녹이는 템플릿입니다.

**템플릿 1: 과정 분해형**

```
[앱 이름]을/를 만들고 싶습니다.

과정을 다음과 같이 나눴습니다:
[1단계]: [설명]
[2단계]: [설명]
[3단계]: [설명]
...

각 단계에서:
[1단계]는 [세부 항목들]
[2단계]는 [세부 항목들]
...

이를 바탕으로 [원하는 기능] 만들어주세요.
```

**템플릿 2: 패턴 활용형**

```
[문제/상황]에서 다음 패턴을 발견했습니다:

시간 패턴:
- [시간대별 특징]

행동 패턴:
- [반복되는 행동]

예외 패턴:
- [특이 케이스]

이 패턴들을 활용하여:
1. [패턴 기반 기능 1]
2. [패턴 기반 기능 2]
3. [패턴 기반 기능 3]

구현해주세요.
```

**템플릿 3: 통합형**

```
[앱 목적]

[분해]
전체를 [N]개 부분으로 나눴습니다:
1. [부분 1] - [설명]
2. [부분 2] - [설명]
...

[패턴]
분석한 패턴:
- [패턴 1]
- [패턴 2]
- [패턴 3]

[기능]
분해와 패턴을 바탕으로:
- [기능 1]: [분해의 어떤 부분을 다루는지]
- [기능 2]: [어떤 패턴을 활용하는지]
- [기능 3]: [통합적 기능]

[추가 요청]
- [스타일, 제약조건 등]
```

### 실습 과제

다음 주제들로 프롬프트를 작성해보세요. 각 과제는 분해와 패턴 인식을 모두 연습하는 기회입니다.

**과제 1: 물 마시기 알림 앱**
- 분해: 하루를 시간대별로
- 패턴: 언제 물을 자주 잊는지
- 기능: 적절한 시간에 알림

**예시 프롬프트:**
```
물 마시기 알림 앱을 만들어주세요.

[분해]
하루를 다음과 같이 나눴습니다:
1. 아침 (07:00-12:00): 기상 후, 출근 시간
2. 점심 (12:00-18:00): 식사 후, 오후 업무 시간
3. 저녁 (18:00-23:00): 퇴근 후, 저녁 시간

[패턴]
제가 물을 잊는 패턴:
- 업무 집중할 때 (오전 10시~12시)
- 회의가 많은 날
- 저녁에 집에 오면 물 대신 커피나 음료수

[기능]
1. 2시간마다 물 마시기 알림
2. 집중 시간대에는 조금 더 자주 알림
3. 저녁에는 "물을 마시세요" 대신 "커피 대신 물 한 잔 어때요?" 메시지
4. 하루 목표량 (2L) 표시
```

**과제 2: 공부 시간 관리 앱**
- 분해: 공부 과정 (과목, 휴식 포함)
- 패턴: 집중 가능 시간, 효율적인 시간대
- 기능: 맞춤 스케줄 제안

**예시 프롬프트:**
```
공부 시간을 관리하는 앱을 만들어주세요.

[분해]
공부 과정:
1. 계획하기 (오늘 할 과목, 시간 배분)
2. 공부하기 (집중 시간, 휴식 시간)
3. 복습하기 (오늘 배운 내용 정리)
4. 평가하기 (계획 대비 실행 확인)

[패턴]
제 공부 패턴:
- 집중 가능 시간: 50분 (이후 집중력 급격히 떨어짐)
- 효율적인 시간대: 아침 (08:00-10:00), 저녁 (20:00-22:00)
- 비효율적인 시간대: 점심 직후 (13:00-15:00)
- 어려운 과목은 아침에, 쉬운 과목은 저녁에 잘 됨

[기능]
1. 50분 공부 + 10분 휴식 타이머
2. 어려운 과목은 아침 시간에 자동 배치
3. 점심 직후는 가벼운 복습 시간으로 제안
4. 하루 목표 시간 달성률 표시
5. 휴식 시간에는 스트레칭 추천
```

**과제 3: 감정 일기 앱**
- 분해: 하루를 시간/활동별로
- 패턴: 어떤 활동이 기분을 좋게/나쁘게 하는지
- 기능: 기분 패턴 분석 및 제안

**예시 프롬프트:**
```
감정을 기록하고 패턴을 분석하는 일기 앱을 만들어주세요.

[분해]
하루를 활동별로 나눔:
1. 아침 루틴 (기상, 준비)
2. 출근/등교 (이동 시간)
3. 오전 활동 (업무, 수업)
4. 점심 시간
5. 오후 활동
6. 저녁 시간 (퇴근 후)
7. 잠들기 전

[패턴]
기분에 영향을 주는 패턴:
- 날씨: 맑은 날 기분 좋음
- 사람: 친구와 만난 날 행복함
- 활동: 운동한 날 기분 상승, 회의 많은 날 스트레스
- 시간: 밤늦게까지 깨어있으면 다음 날 우울함

[기능]
1. 하루에 3번 (아침, 저녁, 잠들기 전) 기분 기록 요청
2. 기분과 활동 연결 (무엇을 했는지 선택)
3. 일주일 단위로 패턴 보여주기
   예: "운동한 날은 평균 행복도가 20% 높아요"
4. 기분 좋아지는 활동 추천
   예: "요즘 우울하시네요. 친구와 만나는 건 어때요?"
```

### 프롬프트 작성 실전 팁

여러분이 직접 프롬프트를 작성할 때 도움이 될 팁들입니다.

**1. 명확한 구조 사용하기**

좋지 않은 예:
```
물 마시는 걸 자꾸 잊어버려서 알림 앱 만들어줘. 
아침, 점심, 저녁에 알려주고, 특히 일할 때 많이 잊어버리니까 
그때 더 자주 알려주면 좋겠어.
```

좋은 예:
```
[목적] 물 마시기 알림 앱

[분해]
- 아침 (07:00-12:00)
- 점심 (12:00-18:00)
- 저녁 (18:00-23:00)

[패턴]
- 업무 시간 (09:00-18:00)에 자주 잊음

[기능]
- 2시간마다 기본 알림
- 업무 시간에는 1시간마다 알림
```

구조가 명확하면 GitHub Copilot이 더 정확하게 이해합니다.

**2. 구체적인 예시 포함하기**

좋지 않은 예:
```
지출을 관리하는 앱 만들어줘.
```

좋은 예:
```
지출을 관리하는 앱 만들어줘.

예를 들어:
- "점심 8,000원" 입력하면 자동으로 식비 카테고리에 저장
- "매주 금요일에 식비가 평소보다 2배 높아요" 같은 패턴 알림
```

구체적인 예시가 있으면 의도를 더 명확히 전달할 수 있습니다.

**3. "왜"를 설명하기**

좋지 않은 예:
```
알림을 오후 3시에만 보내줘.
```

좋은 예:
```
알림을 오후 3시에만 보내줘.
왜냐하면 그 시간이 내가 가장 물을 잊어버리는 시간이거든.
```

이유를 설명하면 더 나은 제안을 받을 수 있습니다.

**4. 제약 조건 명시하기**

좋은 예:
```
간단하고 사용하기 쉽게 만들어주세요.
복잡한 설정은 필요 없어요.
버튼은 크게, 글자는 읽기 쉽게 해주세요.
```

어떤 스타일을 원하는지, 어떤 제약이 있는지 말하면 더 만족스러운 결과를 얻습니다.

### 여러분만의 문제 해결하기

이제 실제로 여러분이 해결하고 싶은 문제를 선택해 프롬프트를 작성해봅시다.

**단계별 가이드:**

**1단계: 문제 선택**
- 여러분이 실제로 불편함을 느끼는 것
- 반복적으로 하는 일
- 자동화하고 싶은 것

예시:
- 매일 약 먹는 것을 잊어버림
- 주차 위치를 기억 못 함
- 식물에 물 주는 시기를 놓침
- 생일을 자주 잊음
- 운동 계획은 세우는데 실행을 못 함

**2단계: 분해하기**
- 그 문제/과정을 단계로 나누기
- 각 단계의 어려운 점 파악하기

**3단계: 패턴 찾기**
- 언제 문제가 발생하는지
- 어떤 조건에서 잘/못 하는지
- 반복되는 실수나 성공 경험

**4단계: 프롬프트 작성**
- [분해], [패턴], [기능] 구조 사용
- 구체적인 예시 포함
- 왜 그렇게 하고 싶은지 설명

**5단계: GitHub Copilot에게 요청**
- 작성한 프롬프트를 GitHub Copilot에게 전달
- 결과를 확인하고 부족한 점 보완
- 필요하면 프롬프트 수정하여 다시 요청

### 실습 중 흔한 실수와 해결법

**실수 1: 너무 크게 나누기**

```
잘못된 분해:
1. 아침 준비
2. 외출
```

이렇게 크게 나누면 각 단계가 여전히 복잡합니다.

```
개선된 분해:
1. 아침 준비
   - 씻기
   - 옷 입기
   - 아침 먹기
2. 외출 준비
   - 준비물 챙기기
   - 집 확인
   - 나가기
```

**실수 2: 패턴이 아닌 우연의 일치**

```
잘못된 패턴:
"화요일마다 지출이 많다" → 단순히 이번 달에 화요일에 약속이 많았을 수도
```

진짜 패턴인지 확인하려면:
- 여러 주, 여러 달 데이터를 봐야 함
- 왜 그런 패턴인지 이유가 있어야 함

```
개선된 패턴:
"주 중반 (화, 수)에 피로가 쌓여서 배달 음식 주문이 늘어난다"
→ 이유가 명확하고, 여러 주 동안 반복됨
```

**실수 3: GitHub Copilot에게 모호하게 요청**

```
모호한 요청:
"편리한 앱 만들어줘"
```

무엇이 편리한지 모릅니다.

```
명확한 요청:
"버튼은 크게 (최소 1cm), 글자는 읽기 쉽게 (최소 14pt),
복잡한 설정 없이 바로 사용 가능한 앱"
```

### 실습 회고 질문

오늘 실습을 마치며 스스로에게 물어보세요:

1. **분해 관련:**
   - 어떤 분해가 가장 어려웠나요? 왜 그랬을까요?
   - 여러분의 분해와 예시의 분해가 어떻게 달랐나요?
   - 다음에는 어떻게 더 잘 분해할 수 있을까요?

2. **패턴 관련:**
   - 어떤 패턴을 발견했나요?
   - 그 패턴이 우연의 일치인지 진짜 규칙인지 어떻게 판단했나요?
   - 발견한 패턴을 어떻게 활용할 수 있을까요?

## 실습 결과 요약

이번 주차에서 우리는 분해와 패턴 인식을 실제로 적용하는 방법을 배웠습니다. 이론을 넘어 실전에서 어떻게 활용하는지 경험했습니다.

### 핵심 내용 정리

**분해의 실전 적용**
- 큰 문제를 작은 단계로 나누는 구체적 방법
- 2-3단계 계층적 분해
- 독립성, 완결성, 적절한 크기, 명확성의 기준
- 실생활 예제: 아침 루틴, 온라인 쇼핑

**패턴 인식의 실전 적용**
- 시간, 공간, 구조, 행동 패턴 구분
- 가계부 데이터에서 패턴 찾기
- 패턴을 활용한 예측과 경고
- 할 일 목록에서 반복 패턴 발견

**GitHub Copilot과의 협업**
- 분해 구조를 프롬프트에 반영하기
- 발견한 패턴을 기능으로 전환하기
- 효과적인 프롬프트 템플릿 3가지
- 실전 예시: 독서 기록, 운동 추적, 습관 형성

### 실습을 통해 배운 것

이번 주 실습에서 여러분은:
✅ 복잡한 과정을 체계적으로 분해할 수 있게 되었습니다
✅ 데이터와 행동에서 패턴을 발견하는 눈을 키웠습니다
✅ 분해와 패턴을 GitHub Copilot에게 전달하는 방법을 익혔습니다
✅ 실생활 문제에 컴퓨팅 사고를 적용하는 경험을 했습니다

### 중요한 깨달음

**분해는 도구입니다**
완벽한 분해는 없습니다. 상황과 목적에 맞게 적절히 나누면 됩니다. 너무 세세하게 나누면 오히려 복잡해지고, 너무 크게 나누면 실행하기 어렵습니다.

**패턴은 숨어 있습니다**
패턴은 데이터를 보기만 해서는 잘 안 보입니다. 시간을 두고 관찰하고, 다양한 각도에서 봐야 합니다. 한번 패턴을 발견하면 많은 것이 자동화되고 예측 가능해집니다.

**프롬프트는 생각의 정리입니다**
GitHub Copilot에게 좋은 프롬프트를 작성하는 과정은 결국 자신의 생각을 정리하는 과정입니다. 명확히 요청하려면 명확히 생각해야 합니다.
