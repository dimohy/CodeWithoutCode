### Chapter 2. 컴퓨팅 사고의 4대 원리
- 컴퓨팅 사고란 무엇인가 - 왜 필요한가
- 분해(Decomposition): 복잡한 시스템을 모듈로
  - 대규모 문제를 관리 가능한 단위로 나누기
  - 모듈 간 의존성 관리
- 패턴 인식(Pattern Recognition): 설계 패턴과 재사용
  - 반복되는 문제 구조 식별
  - 검증된 솔루션 패턴 활용
- 추상화(Abstraction): 인터페이스와 계층 분리
  - 핵심 개념 추출과 세부 구현 분리
  - 계층적 사고와 책임 분리
- 알고리즘적 사고(Algorithmic Thinking): 효율적 문제 해결 절차
  - 단계별 문제 해결 프로세스
  - 최적화와 트레이드오프 고려
- 실습: GitHub Copilot으로 4대 원리 적용해보기
  - 간단한 시스템 설계 문제
  - 각 원리를 적용한 프롬프트 작성
  - 결과 비교 및 분석

### Chapter 3. 고급 컴퓨팅 사고와 AI 시대의 소프트웨어 아키텍처
- 고급 컴퓨팅 사고: 문제 공간과 솔루션 공간
  - 문제 도메인 분석
  - 솔루션 설계 전략
  - 다중 추상화 레벨과 시스템 사고
- 기존 개발 방법론의 한계와 패러다임 전환
- AI 협업 기반 설계 패턴
  - 모듈화 전략
  - 인터페이스 설계
  - 확장 가능한 구조
- 실습: GitHub Copilot을 활용한 시스템 설계 초안 작성
