### 제1부 패러다임의 전환

#### 제1장 프로그래밍 패러다임의 전환점
- 1.1 코딩의 종말, 사고의 시작
- 1.2 바이브 코딩이란 무엇인가
- 1.3 책의 구성 및 학습 목표

#### 제2장 컴퓨팅 사고의 4대 원리
- 2.1 컴퓨팅 사고란 무엇인가 - 왜 필요한가
- 2.2 분해(Decomposition): 복잡한 시스템을 모듈로
- 2.3 패턴 인식(Pattern Recognition): 설계 패턴과 재사용
- 2.4 추상화(Abstraction): 인터페이스와 계층 분리
- 2.5 알고리즘적 사고(Algorithmic Thinking): 효율적 문제 해결 절차
- 2.6 실습: GitHub Copilot으로 4대 원리 적용해보기

#### 제3장 고급 컴퓨팅 사고와 AI 시대의 소프트웨어 아키텍처
- 3.1 고급 컴퓨팅 사고: 문제 공간과 솔루션 공간
- 3.2 기존 개발 방법론의 한계와 패러다임 전환
- 3.3 AI 협업 기반 설계 패턴
- 3.4 실습: GitHub Copilot을 활용한 시스템 설계 초안 작성
